    Troubleshooting: Content Security Policy (CSP)

    Some sites enforce a Content Security Policy that blocks external scripts and fetch requests. If your site has a CSP, the widget may fail silently.

    How to tell if CSP is blocking the widget

    Open your browser's developer tools (F12 or Cmd+Option+I) and check the Console tab. CSP violations look like:

    Refused to load the script 'https://webrung.com/webrung.js' because it
violates the following Content Security Policy directive: "script-src 'self'"

    You may also see blocked requests in the Network tab with status (blocked:csp).

    How to fix it

    Allow webrung.com in two CSP directives: script-src (to load the widget) and connect-src (to fetch sites.json).

    Self-hosting as an alternative

    If you can't modify your CSP, you can self-host the widget:

    1. Download webrung.js and sites.json and place them on your own domain.
    2. Reference your local copy instead: 
      <script src="/webrung.js"></script>

    Since both files are served from your own origin, no CSP changes are needed. Note that sites.json won't auto-update when new members join.

    Start Your Own Webring

    Want to create your own webring? This project is open source. Fork the repository, update sites.json with your ring's name and members, and deploy to GitHub Pages.

    1. Fork the repository Fork on GitHub to get your own copy.
    2. Customize your ring Edit sites.json to set your ring's name, URL, and member sites. Update the CNAME file with your domain (or remove it to use the default GitHub Pages URL).
    3. Enable GitHub Pages Go to your fork's Settings > Pages and deploy from the main branch. Your webring hub will be live automatically.
